StarForge is built around a few core concepts. Understanding these helps you write better layers and debug build issues.

- [Projects](projects/) -- Project files, targets, and remote layers
- [Layers](layers/) -- Layer structure, step fields, override semantics, and `!include`
- [Build Pipeline](build-pipeline/) -- Collect, Execute, and Package stages with caching
- [Variables](variables/) -- Variable substitution, scoping, imports, and exports

For the full list of actions and their fields, see the [Actions Reference](../actions/).
