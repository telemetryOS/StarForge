# Concepts

StarForge is built around a few core concepts. Understanding these helps you write better layers and debug build issues.

- [Projects](projects.md) -- Project files, targets, and remote layers
- [Layers](layers.md) -- Layer structure, step fields, override semantics, and `!include`
- [Build Pipeline](build-pipeline.md) -- Collect, Execute, and Package stages with caching
- [Variables](variables.md) -- Variable substitution, scoping, imports, and exports

For the full list of actions and their fields, see the [Actions Reference](../actions/README.md).
